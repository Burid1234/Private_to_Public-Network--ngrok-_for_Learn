# Private_to_Public-Network_ngrok_for_Learn

#โปรเจคขนาดย่อยนี้ คือการเปลี่ยน “My_Computer” ให้กลายเป็น Sever ที่เป็นโหนดปลายทาง ของ Internet Network System ผ่าน (ngrok)
_________________________________________________________________________________________________________________________

โดยปกติเวลาผมจะสร้าง Web server ขึ้นมาตอนที่ผมนั้นลองฝึกฝนใหม่ๆ การจำลองเว็ปขึ้นมานั้นสามารถเปิดใช้ Web server ได้ในเฉพาะวงLANเดียวกันเท่านั้น
จะไม่สามารถเปิด Web server นี้ผ่านเครือข่ายอื่นได้เลย นอกเหนือจาก เครือข่ายเดียวกันกับ Sever 

แต่โปรเจคขนาดย่อยนี้ผมได้ลองศึกษา นั้นสามารถเปิด Web server หรือ รับส่งข้อมูลข้ามเครือข่ายอินเทอร์เน็ตสาธารณะได้ครับ“WAN”  ไม่จำเป็นต้องในวง“LAN”เดียวกัน
โดยการทำงานประกอบไปด้วย2ส่วนหลักๆคือ
Web Serving Layer และ Network Tunneling Layer

__________________________________________________________________________________________________________________________
**Web Serving Layer** 
คือการสั่งให้ My_Computer เปิดบริการ เพื่อรอรับการติดต่อผ่านโปรโตคอล HTTP เปรียบได้กับเราสร้างห้องขึ้นมาในบ้าน แต่ในสภาวะปกติ ห้องนี้จะถูกซ่อนอยู่หลังกำแพง
(เปรียบคือ “Router”) ทำให้คนที่อยู่นอกบ้าน (เปรียบคือ “Internet”)  มองไม่เห็น

**Network Tunneling Layer (ผ่าน ngrok)** 
คือสิ่งสำคัญของการแก้ไขปัญหา ในขณะที่ ISP Router ให้ Private IP (CGNAT)  ซึ่งเปรียบเสมือนบ้านที่ไม่มีเลขที่บ้านจริงๆบนแผนที่โลก แต่การใช้ ngrok คือการที่ My_Computer วิ่งออกไปสร้างอุโมงค์เชื่อมต่อกับ Server ของ ngrok ที่มี Public IP จริงๆ

**ผลลัพท์ที่ผมต้อการในการศึกษาครั้งนี้คือ** 
เมื่อให้ผู้คนอื่น เข้าที่ลิงก์ของ ngrok ที่สร้างมา ข้อมูลก็จะวิ่งไปที่ Server ของ ngrok ก่อน จากนั้นข้อมูลจะถูกส่งต่อ ผ่านอุโมงค์พิเศษไปที่ My_Compute ที่สร้างไว้ ทำให้ข้อมูลสามารถมุดผ่าน Router และ Firewall เข้ามาถึงไฟล์ใน My_Compute ได้โดยตรง ก็คือผมสามารถเข้าถึง My_Compute ได้ทุกที่

!! แต่วิธีที่เป็น base-on และเป็นพื้นฐานที่สุดคือการ ตั้งค่า config ผ่าน ISP Router โดยตรงในส่วนนี้
เราจะต้องเข้าผ่าน 192.168.1.1 โดยตรงของทาง ตัวแทนเครือข่าย Internet และขั้นตอนจากนั้นคือการ สมัคร THDDNS เพื่อให้สามารถเข้าถึงอุปกรณ์ในบ้าน (เช่น กล้องวงจรปิด NAS, Server, หรือ Router) ผ่านอินเทอร์เน็ตจากภายนอกได้ง่ายๆ โดยใช้ชื่อโดเนมคงที่ แทนการจำหมายเลข IP ที่เปลี่ยนแปลงตลอดเวลา (Dynamic IP) แต่ผมมีเหตุผลและหลายปัจจัยในการที่จะไม่ใช้ base-on นี้และใช้ตัวของ ngrok แทน
